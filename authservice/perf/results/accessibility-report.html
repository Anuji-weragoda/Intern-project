Accessibility Test Report

Date: 2025-11-06   |   Repository: Intern-project (branch: Testing)

Summary

This report summarizes the automated accessibility scan findings and remediation suggestions / changes applied to the admin frontend on 2025-11-06.

Key findings

  1. Insufficient color contrast (WCAG 2.0 AA)

    Text using Tailwind class text-green-600 on white backgrounds had contrast ~3.29:1. Fixed by updating to text-green-900 in source files (Dashboard, Profile, AuditLog). Rebuild required to update generated artifacts.

  2. Select elements missing accessible name (WCAG 2.0 A)

    Several <select> controls had no programmatic name. Fixed by associating <label htmlFor=...> with select id and updating the reusable SelectField component.

  3. Icon-only buttons missing accessible name

    Added aria-label and title for icon-only controls (pagination buttons, modal close buttons).

  4. Horizontally scrollable containers not keyboard-focusable

    Made scroll containers focusable via tabIndex=0 and added aria-label to describe their purpose.

How to verify locally

  1. From repo root, change to admin_frontend and run build or dev server: 
    cd admin_frontend
npm install
npm run build
  2. Run your accessibility scanner (aXe, WAVE, or the tool you used previously) and confirm the earlier issues are resolved.
  3. Manual checks: keyboard navigation (Tab), focusable scroll containers, screen reader announcement of button names and select labels.

<select id="eventTypeFilter" ...> ... </select>

  <X className="w-5 h-5" />

</div>

Recommendations

  • Add automated axe checks to CI to prevent regressions.
  • Prefer design tokens for color choices to ensure accessible contrast by default.
  • Make a small component pattern for icon-only buttons that enforces an accessible name prop.
